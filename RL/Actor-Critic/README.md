# Actor-Critic (On-Policy vs Off-Policy)

This folder provides a **clear comparison** between on-policy and off-policy
actor-critic methods. There are two small tabular examples and a CartPole study
with plots and a GIF, similar to the DQN study in the theorem folder.

## 1. What Actor-Critic Means

We have two components:

- **Actor**: the policy $\pi_\theta(a\mid s)$
- **Critic**: estimates value to reduce variance

The actor is updated using a gradient scaled by a value estimate from the critic.

## 2. On-Policy Actor-Critic (TD(0))

On-policy uses data from the **current policy** only. With value critic $V(s)$:

$$
\delta = r + \gamma V(s') - V(s)
$$

$$
V(s) \leftarrow V(s) + \alpha_v\,\delta
$$

$$
	heta \leftarrow \theta + \alpha_\pi\,\delta\,\nabla_\theta \log \pi_\theta(a\mid s)
$$

Key point: the transition $(s,a,s')$ was generated by the same policy we update.

## 3. Off-Policy Actor-Critic (Replay Buffer)

Off-policy collects data with a **behavior policy** $\mu$ and stores it in a
replay buffer $\mathcal{R}$. We learn a critic $Q(s,a)$ from buffer data, while
the actor uses **current policy actions**, not the buffer actions.

Critic target (expected SARSA style):

$$
\hat y = r + \gamma \sum_{a'} \pi_\theta(a'\mid s') Q(s',a')
$$

$$
Q(s,a) \leftarrow Q(s,a) + \alpha_q(\hat y - Q(s,a))
$$

Actor update (state from buffer, action from policy):

$$
	heta \leftarrow \theta + \alpha_\pi\,Q(s,\tilde a)\,\nabla_\theta\log \pi_\theta(\tilde a\mid s),
\quad \tilde a \sim \pi_\theta(\cdot\mid s)
$$

This is the lecture detail: **actions used for the actor update are not taken
from the replay buffer**.

## 4. CartPole Study Case (Visual)

We use the same custom CartPole dynamics as the DQN study:

- Observation: $[x, x\_dot, theta, theta\_dot]$
- Actions: 0 = left, 1 = right
- Reward: +1 per step (max 500)
- Termination: $|x|>2.4$ or $|theta|>12^\circ$ or max steps

The script produces:

- Training curves: reward, actor loss, critic loss, policy entropy
- Trajectory plot: $theta$, $theta\_dot$, $x$, $x\_dot$
- GIF: CartPole animation for the learned policy

## 6. How to Compare

When you compare the outputs, focus on:

- **Stability**: on-policy is more stable but can be sample-inefficient.
- **Sample reuse**: off-policy improves data efficiency using replay.
- **Bias vs variance**: off-policy uses a broader state distribution and can
	introduce bias, but often speeds up learning.
